Node Setup Guide - OS-Specific Installation

Bitcoin Node Installation on Linux

Prerequisites

Ensure you have sudo access and your system is up to date.

Step 1: Update System

# Update package lists and upgrade system
sudo apt update && sudo apt upgrade -y

# Install essential tools
sudo apt install curl wget gnupg2 software-properties-common -y

Step 2: Download Bitcoin Core

# Create bitcoin directory
mkdir -p ~/bitcoin && cd ~/bitcoin

# Download Bitcoin Core (replace with latest version)
wget https://bitcoin.org/bin/bitcoin-core-25.0/bitcoin-25.0-x86_64-linux-gnu.tar.gz

# Download signature file
wget https://bitcoin.org/bin/bitcoin-core-25.0/SHA256SUMS.asc

Step 3: Verify and Install

# Verify the download
sha256sum --check SHA256SUMS.asc --ignore-missing

# Extract and install
tar -xzf bitcoin-25.0-x86_64-linux-gnu.tar.gz
sudo install -m 0755 -o root -g root -t /usr/local/bin bitcoin-25.0/bin/*

Next: Configure your bitcoin.conf file and start the node.

Bitcoin Node Installation on macOS

Prerequisites

Install Homebrew if you haven't already. This makes package management much easier.

Step 1: Install Homebrew (if needed)

# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install essential tools
brew install curl wget gnupg

Step 2: Download Bitcoin Core

# Create bitcoin directory
mkdir -p ~/bitcoin && cd ~/bitcoin

# Download Bitcoin Core for macOS
curl -O https://bitcoin.org/bin/bitcoin-core-25.0/bitcoin-25.0-osx64.tar.gz

# Download signature file
curl -O https://bitcoin.org/bin/bitcoin-core-25.0/SHA256SUMS.asc

Step 3: Verify and Install

# Verify the download
shasum -a 256 --check SHA256SUMS.asc --ignore-missing

# Extract the archive
tar -xzf bitcoin-25.0-osx64.tar.gz

# Move to Applications (optional)
sudo cp -R bitcoin-25.0/ /Applications/Bitcoin-Core/

Note: You may need to allow the application in System Preferences > Security & Privacy.

Bitcoin Node Installation on Windows

Prerequisites

Run PowerShell as Administrator for some commands. Windows Defender may flag the software initially.

Step 1: Download Bitcoin Core

# PowerShell commands
# Create bitcoin directory
New-Item -ItemType Directory -Path "$env:USERPROFILE\bitcoin" -Force
Set-Location "$env:USERPROFILE\bitcoin"

# Download Bitcoin Core
Invoke-WebRequest -Uri "https://bitcoin.org/bin/bitcoin-core-25.0/bitcoin-25.0-win64.zip" -OutFile "bitcoin-25.0-win64.zip"

Step 2: Verify and Extract

# Verify hash (optional but recommended)
Get-FileHash bitcoin-25.0-win64.zip -Algorithm SHA256

# Extract the archive
Expand-Archive -Path "bitcoin-25.0-win64.zip" -DestinationPath "."

Step 3: Configure Windows Defender

Add the Bitcoin Core directory to Windows Defender exclusions:

  1. Open Windows Security
  2. Go to Virus & threat protection
  3. Add exclusion for the bitcoin folder
  4. Run bitcoin-qt.exe or bitcoind.exe

Tip: Consider creating a desktop shortcut for easy access.

Configuration Options

Basic Configuration

Essential settings to get your node running:

Create bitcoin.conf

Create this file in your Bitcoin data directory:

  • Linux: ~/.bitcoin/bitcoin.conf
  • macOS: ~/Library/Application Support/Bitcoin/bitcoin.conf
  • Windows: %APPDATA%\Bitcoin\bitcoin.conf
# Basic bitcoin.conf
# Network
port=8333
listen=1
maxconnections=125

# Storage
datadir=~/.bitcoin
dbcache=450

# Logging
debug=0
logtimestamps=1

Advanced Configuration

Fine-tune your node for optimal performance:

Memory Pool Settings

# Memory pool configuration
maxmempool=300
mempoolexpiry=72
incrementalrelayfee=0.00001000

Network Optimization

# Network settings
maxuploadtarget=5000
blocksonly=0
peerbloomfilters=1
permitbaremultisig=1

Performance Tuning

# Performance settings
par=4
checkblocks=6
checklevel=3
assumevalid=0

Important Notes

  • Adjust par based on your CPU cores
  • Higher dbcache values speed up initial sync
  • Monitor bandwidth usage with maxuploadtarget

Security Configuration

Secure your node against common threats:

RPC Security

# RPC authentication
rpcauth=nodeuser:7d9ba5ae63c3d4dc30583ff4fe65a67e$9e3634e81c11659e3de036d0bf88f89cd169c1039e6e09607562d54765c649cc
rpcbind=127.0.0.1
rpcallowip=127.0.0.1
rpcport=8332

Network Privacy

# Tor configuration (optional)
proxy=127.0.0.1:9050
onlynet=onion
listen=0
listenonion=1

Firewall Rules

Configure your firewall to allow Bitcoin traffic:

# UFW (Ubuntu/Debian)
sudo ufw allow 8333/tcp
sudo ufw enable

# iptables
sudo iptables -A INPUT -p tcp --dport 8333 -j ACCEPT

Security Checklist

  • ✓ Use strong RPC authentication
  • ✓ Bind RPC to localhost only
  • ✓ Configure firewall rules
  • ✓ Consider Tor for privacy
  • ✓ Regular security updates
